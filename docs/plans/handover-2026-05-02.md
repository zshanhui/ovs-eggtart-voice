## 2. 核心结论

**TRT-Edge-LLM C++ runtime 存在精度 bug，导致 TTS 和 ASR 引擎均产出垃圾数据。**

| 测试组合 | 结果 |
|---------|------|
| 旧 ONNX + TRT-Edge-LLM 引擎 | 垃圾 (ASR: "嗯。", TTS: 噪声) |
| 旧 ONNX + BF16 重建引擎 | 垃圾 |
| 新 ONNX (WSL2 导出) + BF16 引擎 | 垃圾 |
| 相同引擎 + 旧 `qwen3_speech_engine` runtime | **正常** |

推理: 不是 ONNX 格式、引擎构建精度问题。问题在 TRT-Edge-LLM 的 `LLMEngineRunner` / 采样 / KV cache / multimodal audio bridge 逻辑中。
